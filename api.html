Инструкции по работе с API для интеграции

Подробные инструкции по работе с API и примеры интеграции с веб-сайтом Tiger SMS. Все необходимые данные для эффективной интеграции с сервисом Tiger SMS и получения СМС в автоматическом режиме.

Наш API полностью совместим с сайтами конкурентов

Все запросы должны направляться на https://tiger-sms.com/stubs/handler_api.php ОТПРАВИТЬ или ПОЛУЧИТЬ запрос

Все запросы должны иметь ключ API в качестве параметра 'api_key'

  • Получить баланс

    https://tiger-sms.com/stubs/handler_api.php?api_key=WfGRI7ELx6vqbDpNWKVryL1MQoimlZcL&action=getBalance

    Параметры:

    $api_key - Ваш API ключ

    Ответ:

    ACCESS_BALANCE:'баланс аккаунта'

    ВОЗМОЖНЫЕ ОШИБКИ:

    BAD_KEY - недопустимый ключ API

  • Получить номер телефона

    https://tiger-sms.com/stubs/handler_api.php?api_key=WfGRI7ELx6vqbDpNWKVryL1MQoimlZcL&action=getNumber&service=$service&country=$country&ref=$ref

    Параметры:

    $api_key - Ваш API ключ

    $service - service смотреть таблицу

    $country - номер страны смотреть таблицу

    $ref - Реферал ID

    Получая номер через API, вы соглашаетесь с правилами проекта

    ВОЗМОЖНЫЕ ОШИБКИ:

    BAD_ACTION - неправильное действие

    BAD_SERVICE - неправильнрое название сервиса

    BAD_KEY - недопустимый ключ API

    BAD_REF - неверный реферал ID

  • Получить смс код

    https://tiger-sms.com/stubs/handler_api.php?api_key=WfGRI7ELx6vqbDpNWKVryL1MQoimlZcL&action=getStatus&id=$id

    Параметры:

    $api_key - Ваш API ключ

    $id - id активации

    ОТВЕТ:

    STATUS_WAIT_CODE - Ожидание СМС

    STATUS_WAIT_RESEND - ожидание следующего SMS

    STATUS_CANCEL - активация отменена

    STATUS_OK: 'код активации' - полученный код

    ВОЗМОЖНЫЕ ОШИБКИ:

    BAD_KEY - недопустимый ключ API

    BAD_ACTION - неправильное действие

    NO_ACTIVATION - неверный id активации

  • Изменения статуса активации

    https://tiger-sms.com/stubs/handler_api.php?api_key=WfGRI7ELx6vqbDpNWKVryL1MQoimlZcL&action=setStatus&status=$status&id=$id

    Параметры:

    $api_key - Ваш API ключ

    $id - id активации

    $status - статус активации

    • 1 - сообщить о готовности номера (SMS отправлено на номер)
    • 3 - запросите другой код (бесплатно)
    • 6 - полная активация *
    • 8 - сообщите, что номер был использован, и отмените активацию

    * если был статус "код получен" - помечает его успешно и завершается, если была "подготовка" - удаляет и помечает ошибку, если был статус "ожидание повторной попытки" - переводит активацию в ожидание SMS

    ** Невозможно изменить статус активации, для которого был выбран метод проверки по вызову, если номер уже поступил

    Простая логика хронологии API:

    Получить номер с помощью метода getNumber после этого доступны следующие действия:

    • 8 - Отмените активацию (если номер вам не подходит)
    • 1 - Сообщить, что SMS было отправлено (необязательно)
    • К активации со статусом 1:
    • 8 - Отменить активацию сразу после получения кода:
    • 3 - Запросите другое SMS
    • 6 - Подтвердите SMS-код и завершите активацию
    • К активации со статусом 3:
    • 6 - Подтвердите SMS-код и завершите активацию
    ОТВЕТ:

    ACCESS_READY - телефон готов к получению SMS

    ACCESS_RETRY_GET - ожидание нового смс

    ACCESS_ACTIVATION - сервис был успешно активирован

    ACCESS_CANCEL - активация отменена

    ВОЗМОЖНЫЕ ОШИБКИ:

    NO_ACTIVATION - неверный id активации

    BAD_SERVICE - неправильнрое название сервиса

    BAD_STATUS - неправильный статус

    BAD_KEY - недопустимый ключ API

    BAD_ACTION - неправильное действие
